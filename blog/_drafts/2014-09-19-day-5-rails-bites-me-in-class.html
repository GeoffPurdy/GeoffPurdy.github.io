--- layout: post title: 'Day 5: Rails bites me in the class' date: '2014-09-19T20:31:00.000-07:00' author: geoff purdy tags: modified_time: '2017-03-26T05:00:52.547-07:00' blogger_id: tag:blogger.com,1999:blog-1206336038417661147.post-6851189858679219125 blogger_orig_url: http://geoffreypurdy.blogspot.com/2014/09/day-5-rails-bites-me-in-class.html --- Last day of week 1.  I am REALLY enjoying the class thus far.  Something about this style of learning appeals to me.  I have spent a great deal of time coding but haven't been bored.

Today was a little frustrating.  I realized that I have never programmed in a language that will allow declaration of variables as liberally as ruby.  And the errors are manifested at run time.  Typical error for me is something like:
@things = new Array(2)
@thing[0] = "Thing 1"
@thing[1] = "Thing 2"

See the problem?  I named the array in the plural, but used the singular accessing the individual elements.  Bad habit of mine.  That gets caught pretty quickly in the other languages I have used.  Ruby happily runs the code then complains you are calling method []= on nil.  Because you created a brand new object when misspelling the array name.  Easy fix, but also easy to overlook.

Method return types have also bitten me a few times.  I suspect a fancier IDE might be a big help with this.
